## 核心理念

- **原子化**: 一个概念一个文件
- **流重于囤**: Journal 记录上下文，Topics 沉淀知识
- **链接即组织**: 用双向链接连接概念
- **现在整理**: 强制直接分类，拒绝堆积

---

## 文件夹结构

```
00-Inbox/        ← 临时捕获（24h内清空）
10-Journal/      ← 时间流：原始想法
20-Topics/       ← 主题网：原子知识
30-Projects/     ← 项目（有截止日期）
40-Sources/      ← 原始资料元信息
90-Meta/         ← 模板、指南、索引
```

---

## 工作流

### 新知识流转

```
[新信息] → [00-Inbox] → [10-Journal] → [20-Topics] → [更新 MOC]
  立即       24h内         理解后         立即
```

### 各区域职责

| 区域 | 存放内容 | 更新时机 |
|------|----------|----------|
| Inbox | 原始片段、快速记录 | 发生时立即 |
| Journal | 当日上下文 | 每天结束时 |
| Topics | 整理后的原子知识 | 理解后 |
| MOC | 领域索引 | 创建新笔记时 |

## MOC 维护方法

### 什么是 MOC

MOC (Map of Content) 是领域索引文件，帮助导航大量原子笔记。

### 如何维护

1. **创建原子笔记后立即更新 MOC**
2. 打开对应领域的 MOC 文件（如 `20-Topics/AI/AI-MOC.md`）
3. 在合适分类下添加链接：`[[新概念]] - 一句话说明`
4. 保存并提交

### MOC 模板

```markdown
# AI 领域索引

## 基础概念
- [[llm]] - 大语言模型

## 进阶架构
- [[moe]] - 混合专家模型

## 工具/框架
- [[pytorch]] - 深度学习框架
```

## Claudian 自动化

### 触发方式

在 Inbox 文件中输入：`/sort-out-notes`

### 功能

Claudian 会：
1. 读取 Inbox 内容
2. 智能分类建议
3. 询问新概念归属
4. 自动创建/更新笔记
5. 更新相关 MOC
6. 清空 Inbox

### 决策点

以下情况会询问用户：
- 新概念应该放在哪个主题文件夹
- 是否需要创建概念间的链接
- 是否需要深度处理某条内容

## 命名规范

| 类型 | 文件名 | YAML 标题 |
|------|--------|-----------|
| 原子笔记 | 英文小写 + 连字符 | 中文全称 |
| Daily Note | `YYYY-MM-DD.md` | - |
| Sources | 英文小写 | 资料原名 |
| MOC | 大写缩写 + `-MOC` | 中文领域名 |

### 示例

- 文件: `20-Topics/AI/transformer.md`
- 内容: `title: Transformer架构`

## 模板使用

### 创建原子笔记

1. 复制 `90-Meta/templates/atom-topic.md`
2. 粘贴到新文件
3. 填充内容

### 快捷键（Obsidian）

- 快速创建 Daily Note: `Ctrl+Alt+D` (需插件配置)
- 打开命令面板: `Ctrl+P`

## 故障排除

### 问题：不确定如何分类新概念

**解决**: 暂时放在 `20-Topics/concepts/`，当周回顾时再调整。

### 问题：Inbox 堆积太多

**解决**: 立即触发 `/sort-out-notes`，强制整理。

### 问题：双向链接失效

**解决**: 检查文件名拼写，Obsidian 链接区分大小写。
