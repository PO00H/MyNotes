# 类型转换 Cast

## 一句话定义

> [!abstract]
> Cast To = ==把一个泛泛的引用，按某个具体类型"验证 + 转换"==。成功 = 它确实是这个类（并给你该类的引用）；失败 = 不是。所以它既能**筛类型**，又能**拿到专属能力**。

## 📖 要点

- **两大用途**：
	1. ==筛类型==：判断"它到底是不是某个类"（过滤）。
	2. ==拿专属能力==：成功后能访问那个具体类的专属变量/函数（向下转型 = 解锁能力）。
- **为什么"转换"能拿来"检测"**：
	- 安全转换**必须先验证类型** → 内置"是/不是"检查 → ==成功/失败的分叉被借来当检测==。
	- 类比：钥匙能不能转动，本身就告诉你"是不是对的锁"。
- **C++ 对照**：`Cast<T>(p)` ≈ `dynamic_cast`，失败返回 `null`；`if(Cast 成功)` 即类型判断。
- **只想检测、不取引用**：可用 `== Get Player Character` 或 `Actor Has Tag`，更轻。

> [!tip] 门检测玩家的例子
> `Begin Overlap → Cast To BP_FirstPersonCharacter → 成功才开门`。只有玩家是这个类，子弹/道具 Cast 失败 → 被挡在门外。（见 [[被动碰撞检测 事件]]）

> [!warning] Cast 的代价 → 引出 [[接口 BPI]]
> Cast 会让蓝图==硬依赖==那个具体类。要认门、认箱子、认 NPC…就得 Cast 一堆类型，节点爆炸、互相绑死。==Cast 多了就该换 [[接口 BPI]] 解耦。==
