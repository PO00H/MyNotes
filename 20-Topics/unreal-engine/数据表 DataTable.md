# 数据表 DataTable（含结构体 Struct）

## 一句话定义

> [!abstract]
> 数据表（DataTable）= ==一张 Excel 式的表格资产，每行一条配置数据==。先用**结构体 Struct** 定义"每行有哪些列"，再建表填行，运行时按行名查表。改表就能改游戏，不用动蓝图——这就是==数据驱动==。

## 📖 制作流程（顺序不能反）

### 第 1 步：先建结构体 Struct（定义"列"）

![[Pasted image 20260604120408.png]]

- **结构体 = 把多个变量打包成一个自定义类型**（呼应 [[结构体类型]]，那里是内置的 Vector/Rotator，这里是你自己造的）。
- 内容浏览器右键 → 蓝图 → **结构体(Structure)**。
- 往里加字段 = 表格的列。例：武器表 → `名称(String)` + `伤害(Float)` + `模型(StaticMesh)` + `图标(Texture)`。

![[Pasted image 20260604120433.png]]

> [!warning] 为什么必须先有结构体
> ==DataTable 的"每一行"就是一个结构体实例==。没有结构体，表不知道该有哪些列——所以**先 Struct 后 DataTable**，顺序反了建不出来。

### 第 2 步：基于结构体建 DataTable

![[Pasted image 20260604120648.png]]

- 内容浏览器右键 → 杂项 → **数据表(Data Table)** → 弹窗里**选你刚建的结构体**。

![[Pasted image 20260604120720.png]]

- 进表后**逐行填数据**，每行起一个唯一的 **Row Name**（行名 = 这条数据的 ID）。

### 第 3 步：运行时查表

- 节点 **`Get Data Table Row`**：传入表 + Row Name → 输出该行（一个结构体）。
- 再用 `Break` 把结构体拆出各字段用（呼应 [[蓝图引脚拆分与合并]]）。

## 📖 为什么用数据表

- ==数据和逻辑分离==：100 把武器的数值放表里，蓝图只写"读表→生成"一套逻辑。
- 改数值 = 改表格，**不碰蓝图、不重编译**，策划也能改。
- 你见过的 [[GamePlay 框架]] 里 `BP_ShooterPickup` 就是用武器 DataTable 决定生成哪把枪。

> [!tip] 何时该上数据表
> 当你发现"一堆同类东西，只是数值不同"（武器/敌人/物品/关卡配置）——就该把它们抽进一张 DataTable，而不是做一堆蓝图。
