# 蓝图注释与整理

## 一句话定义

> [!abstract]
> 给蓝图加可读性的手段不止"选中按 `C` 的大框"。分两类：==写说明==（气泡 / Tooltip）和==让结构自解释==（折叠成函数 / 理线 / 命名）。好蓝图靠后者为主。

## 📖 A. 写说明类（解释"是什么 / 为什么"）

| 方式 | 怎么做 | 适合 |
|---|---|---|
| **节点气泡** | 悬停节点 → 右上角小气泡图标 → 输入文字 | 解释==某一个节点== |
| **注释框 (C)** | 选中多个节点 → 按 `C` | 框住==一整段逻辑== |
| **变量/函数 Tooltip** | Details 面板填 `Tooltip / Description` | 悬停提示，==不占图面== |

## 📖 B. 结构自解释类（比写注释更高级）

> [!tip] 核心心法
> ==注释会过期，结构不会。== 命名好、折叠好的图本身就是文档。

- **折叠成函数 / 宏**：选中一坨 → 右键 → `Collapse to Function`，乱线变成一个有名字的节点（如 `OpenDoor`），名字即注释，还能复用。
- **Reroute 节点**：双击连线加"拐点(Knot)"，把交叉线理顺。
- **描述性命名**：`IsDoorOpen` 而非 `bool1`；自定义事件/函数都起明确名字。

## 📖 C. 注释框 (C) 进阶

- **配色**：选中框 → Details 改颜色，==按用途配色==（绿=输入 / 红=危险 / 蓝=UI），一眼分区。
- **嵌套 + 字号**：框里可套框，标题字号可调。
- **整体移动**：拖框时内部节点一起走（可在 Details 关掉）。
- **书签 Bookmarks**：图表工具栏书签图标，大图里快速跳转某区域。

## ⚡ 速查

| 想干嘛 | 用哪个 |
|---|---|
| 解释某一个节点 | 节点气泡 |
| 解释变量/函数用途 | Tooltip |
| 框住一段逻辑 | 注释框 C + 配色 |
| 一坨逻辑想收起来 | 折叠成函数 |
| 线太乱 | Reroute |
| 大图找不到地方 | 书签 |

> [!tip] 实战取舍
> 一两个节点 → 气泡；一整段（如"开门时间线"）→ C 框 + 配色；复杂到看不懂 → 别加注释，==直接折叠成函数==。
