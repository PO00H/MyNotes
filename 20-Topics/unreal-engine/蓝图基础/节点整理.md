# 节点整理

## 一句话定义

> [!abstract]
> 把一坨节点收拾干净的三种"折叠"法 + 一个建变量快捷键。和 [[蓝图注释与整理]] 是一对（那篇讲注释，这篇讲折叠/复用）。

## 📖 折叠三法（选中节点 → 右键）

| 方式 | 本质 | 复用性 |
|---|---|---|
| **折叠成 Node** | ≈大括号作用域，只是收起来看着清爽 | ❌ 复用要**复制**（代码量不减） |
| **折叠成函数** | 真正的函数 | ✅ 真复用、功能全（可 Pure、限定参数类型、多用途） |
| **折叠成宏 Macro** | 格式上的复用，像 C++ 宏替换 | ✅ 可复用，且能**多出入口**（函数只能单出口） |

- 选口诀：**只想整理 → Node；要复用 → 函数；要多出入口的流程工具 → 宏**（FlipFlop、ForEach 本身就是系统宏）。
- 选节点技巧：按住 `Ctrl` 点选 = 增/删选中。

## ⚡ Promote to Variable（建变量快捷键）

- 从任意引脚**拖出 → Promote to Variable**：按类型自动建变量 + 自动给 set。
- 省去"手动建变量 + 看类型"——尤其类型一眼看不出时超好用。

> [!tip] 实战
> note6.8 里"开门前先存目标方向"就是拖出点乘结果 → Promote to Variable，避免开门中途转向的 bug。
