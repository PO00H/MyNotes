# 自定义事件 CustomEvent

## 一句话定义

> [!abstract]
> 自定义事件 = ==你自己定义、自己命名的事件==。用法像函数（一段可复用的逻辑，可带输入参数），但==没有返回值==。它是给一段逻辑起名、被触发、或做异步的入口。

## 📖 要点

![[Pasted image 20260605101320.png]]

- **建法**：事件图表右键 → `Add Custom Event` → 命名（如 `OpenDoor`）。
- **像函数一样用**：可加输入参数、在本蓝图里直接调用、把逻辑命名复用。
- **和函数的关键区别**：

| | 函数 Function | 自定义事件 Custom Event |
|---|---|---|
| 返回值 | ==有== | ==无== |
| 执行 | 必须同一帧内返回 | 可跨帧 |
| 能放 Delay / Timeline 吗 | ❌ 不能 | ✅ 能 |

![[Pasted image 20260605101329.png]]

> [!tip] 最实用的判断
> ==需要 [[Delay 延迟]] / [[Timeline 时间轴]] 这种"会跨帧等待"的节点 → 必须用自定义事件，不能用函数。== 因为函数要求当帧算完返回。开门（带 Timeline）就得用事件，不能用函数。

- **常被谁触发**：输入事件、Overlap 事件、[[接口 BPI]] 的无返回值函数、Event Dispatcher。
