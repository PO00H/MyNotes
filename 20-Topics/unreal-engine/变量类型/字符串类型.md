# 字符串类型（FString / FName / FText）

## 一句话定义

> [!abstract]
> UE 有三种"字符串"，按==要不要改、要不要给玩家看==来选：FName 当 ID、FString 做运算、FText 给玩家。

## 📖 要点

| 类型 | 特点 | 典型用途 |
|---|---|---|
| **FName** | 底层 ==Hash 表==存储，查找快，基本不可变 | ID、资源名、标签、骨骼名 |
| **FString** | ==可动态修改==，功能最多，但开销大、性能低 | 临时拼接 / 运算的文本 |
| **FText** | Key-Value，支持==多语言本地化== | 显示给玩家的一切文字 |

- ==性能==：FName（最快）> FString（最重）。
- ==给玩家看的文字一律用 FText==，方便日后做本地化。
- 三者之间可相互转换（如 FString ↔ FName），但转换有成本，别滥用。
