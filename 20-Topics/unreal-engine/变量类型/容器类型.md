# 容器类型（Array / Set / Map）

## 一句话定义

> [!abstract]
> 容器 = ==把一组同类数据装进一个变量==。要有序可重复用 Array，要去重用 Set，要"键→值"查表用 Map。

## 📖 要点

| 容器 | 特点 | 典型用途 |
|---|---|---|
| **Array** 数组 | ==有序==、可重复、按下标访问（[0],[1]…） | 敌人列表、物品栏、路径点 |
| **Set** 集合 | ==自动去重==、无序 | 已解锁成就、不重复的标签 |
| **Map** 映射 | ==键 → 值==成对（字典） | 玩家名→分数、ID→数据 |

- ==选择口诀==：要顺序/可重复 → Array（最常用）；只在乎"有没有/不重复" → Set；要用一个东西查另一个 → Map。
- **Array 高频操作**：Add（加）、Remove、Length（长度）、Get [index]、For Each Loop（遍历）。
- 容器里装的可以是任何类型：`Array<Actor>`、`Array<Vector>`、`Map<String, Integer>`。

> [!warning] 易错点
> - **下标越界**：访问 `Array[5]` 但长度只有 3 会报错；遍历优先用 For Each。
> - **Array 是有序、Set/Map 无序**：需要稳定顺序别用 Set。
