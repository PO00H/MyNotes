# 输入系统（Enhanced Input）

## 一句话定义

> [!abstract]
> 增强输入（Enhanced Input）= UE 的新版输入系统。核心是==把"按哪个键"和"触发什么动作"解耦==：动作写在 [[Input Action]]（做什么），按键绑定写在 [[Input Mapping Context]]（哪个键触发），运行时还能整张映射表换着挂。

## ⚡ 三步用起来

1. **启用插件**：`Edit → Plugins` 搜 `Enhanced Input` → 勾选 → 重启。

![[Pasted image 20260603103119.png]]

2. **建两个资产**（内容浏览器右键 → `Input`）：
	- [[Input Action]]（IA）：定义一个动作（如"跳跃"）。
	- [[Input Mapping Context]]（IMC）：把按键绑到 IA（如 Space → 跳跃）。

![[Pasted image 20260603103126.png]]
3. **挂载 + 响应**：
	- 在 PlayerController / Pawn 里把 IMC 加进去（`Add Mapping Context`）。
	- 蓝图里用 `EnhancedInputAction` 节点监听 IA 的事件（见 [[Input Action]]）。

## 📖 教学（核心原理）

### 1. 为什么不用旧输入

- 旧版（Project Settings 里配 Action/Axis Mapping）==键和动作写死、难复用、不能运行时换==。
- 增强输入==解耦==：同一个 IA 给键盘/手柄/触屏各绑一套；进车/下车换不同 IMC，输入方案随场景切换。

### 2. 两个资产的分工

- ==IA = 做什么==（抽象动作，带一个"值类型"：bool / 1D / 2D 等）。
- ==IMC = 哪个键触发==（一张映射表，把具体按键连到 IA）。
- 一句话：**IA 是"动词"，IMC 是"键位表"。**

### 3. Triggers 与 Modifiers 的生效范围（关键）

- ==写在 IA 里 → 对所有用它的按键都生效==（全局规则）。
- ==写在 IMC 的某个按键下 → 只对这个键生效==（局部规则）。
- 这套"分层覆盖"就是增强输入灵活的根源。
