## ⚡ 两条链路（概念）

| 链路 | 输入 | Input Action | 驱动节点 | 结果 |
|---|---|---|---|---|
| **移动** | WASD | IA_Move（Axis2D） | Add Movement Input | 角色位移 |
| **视角** | 鼠标 X/Y | IA_Look（Axis2D） | Add Controller Yaw / Pitch Input | 转头 / 转身 |

> 两条链==结构完全一样==：键/鼠标 → 一个 Axis2D 的 [[Input Action]] → 蓝图 `Triggered` 事件里读 `Action Value`（Vector2D）→ 喂给驱动节点。

## 📖 教学（核心原理）

### 1. 为什么是"两条独立链路"

- ==移动和视角互不干扰==：各自一个 IA、各自一条蓝图链。
- 共同套路：`EnhancedInputAction` 节点的 `Triggered` → 取出 `Action Value`（一个 [[结构体类型|Vector2D]]）→ 交给对应驱动节点。

### 2. WASD 怎么变成"一个方向"

- 移动只需要一个 ==2D 方向==（前后 + 左右），所以用一个 Axis2D 的 IA_Move，而不是 4 个动作。
- 4 个键合并成一个 2D 向量，靠的就是 [[Input Action|Modifiers]]：
	- ==Negate==：让 S（后）、A（左）成为正方向的反向。
	- ==Swizzle==：把某些键的输入从 X 轴挪到 Y 轴。
- 这正是之前学 Modifiers 时埋的用法——回看 [[Input Action]]。

### 3. 移动为什么要"相对视角"

- ==按 W 是朝"镜头前方"走，不是世界正前方==。
- 所以 Add Movement Input 的方向不是写死的，而是==根据当前视角（控制器朝向）算出的"前 / 右"方向==。
- 否则转了视角后按 W 还往老方向走，手感就错了。

### 4. 鼠标 → 视角

- 鼠标的移动量是个 2D 值：==X 控制 Yaw（左右转身）、Y 控制 Pitch（上下抬头）==。
- 直接喂给 Add Controller Yaw / Pitch Input，转动的是 [[Controller]] 的视角朝向。

> [!warning] 易错点
> - **转了视角后 WASD 方向乱** → 移动方向没基于控制器朝向算"前/右"。
> - **上下看能翻过头** → Pitch 通常要限制范围（Clamp），别让镜头翻跟头。
> - **鼠标上下方向反了** → 在 IMC 里给鼠标 Y 加/去 Negate 调整。
> - **WASD 只有两个方向能动** → IMC 里 Negate / Swizzle 没配全。
