# RAG-检索增强生成

**Retrieval-Augmented Generation**，通过外部知识检索增强大模型生成能力的技术架构。

## 一句话定义

RAG 让大模型在回答问题前先「查资料」，结合检索到的相关信息生成更准确、可追溯的回答。

## 核心原理

```
用户查询 → 检索模块 → 外部知识库 → 相关性文档
                              ↓
用户查询 + 检索结果 → 大模型 → 生成回答
```

### 三个关键组件

| 组件 | 功能 | 技术要点 |
|------|------|----------|
| **检索器 (Retriever)** | 从知识库找相关文档 | 向量相似度、关键词匹配、混合检索 |
| **知识库 (Knowledge Base)** | 存储结构化/非结构化数据 | 向量数据库、文档分块、索引优化 |
| **生成器 (Generator)** | 整合信息生成回答 | 上下文窗口、提示工程、引用格式 |

### 工作流程

1. **索引阶段**：文档分块 → Embedding → 存入向量数据库
2. **检索阶段**：查询向量化 → 相似度搜索 → Top-K 召回
3. **生成阶段**：拼接查询+检索结果 → LLM 生成 → 输出带引用回答

## 应用场景

- **企业知识问答**：基于内部文档的客服/助手
- **专业领域助手**：法律、医疗、金融等需要精准知识的场景
- **幻觉抑制**：通过外部知识约束模型输出，减少编造
